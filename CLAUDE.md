# 가족 가계부 웹앱 — Claude Code 작업 안내

너는 이 앱을 **직접 편집하고 GitHub Pages에 배포**하는 역할이야. 부부(이우승·송혜림)가 함께 쓰는 가계부고, 데이터는 구글 스프레드시트에 저장돼. 앱은 이미 완성·검증된 상태이고, 앞으로는 **항목/카드/사용자 추가, 입력 필드 수정** 같은 자잘한 변경을 네가 바로 하게 될 거야.

---

## 1. 파일 구조 (빌드 없음)

- `index.html` — 앱 전체. **이 파일만 편집하면 된다.** 상단 `<x-dc>` 안에 화면(HTML 템플릿)이 있고, 하단 `class Component extends DCLogic { ... }` 안에 로직이 있다.
- `support.js` — 이 앱을 구동하는 런타임. **절대 편집하지 마.** `index.html`이 `<script src="./support.js">`로 불러오니 저장소에 반드시 같이 있어야 한다.
- `apps-script/Code.gs` — 구글 시트 백엔드(이미 배포됨, 참고용). 시트 저장 규칙을 바꿀 때만 건드린다.

**중요:** 빌드/번들/npm 과정이 전혀 없다. `index.html`을 고치고 커밋·푸시하면 GitHub Pages에 그대로 반영된다.

### 이 템플릿 문법 규칙 (support.js 런타임)
- `{{ 이름 }}` 은 **점 표기 경로만** 넣는 자리다. `{{ a + b }}` 같은 수식 금지 — 계산은 아래 `renderVals()`에서 하고 이름으로 노출한다.
- 반복은 `<sc-for list="{{ items }}" as="item">…</sc-for>`, 조건은 `<sc-if value="{{ flag }}">…</sc-if>`.
- 스타일은 전부 인라인 `style="..."`. 클래스 CSS 쓰지 마.
- 이벤트는 `onClick="{{ 핸들러 }}"` 처럼 값 전체를 넣는다.
- 새 데이터를 화면에 쓰려면 `renderVals()`의 `return { ... }` 에 이름으로 넣고 템플릿에서 `{{ 이름 }}` 으로 참조.

## 2. 자주 하게 될 변경 — 정확한 위치

로직 클래스 맨 위 `state` 안 (대략 203~205줄):

```js
const users = ['이우승', '송혜림'];
const cats = ['그로서리', '외식', '생활용품', '통신', '전기비', '기타'];
const cards = ['CSP(WS)', 'Bonvoy', 'C.Freedom'];
```

- **생활비 항목 추가/삭제** → `cats` 배열만 수정. (예: `'문화'` 추가)
  - 항목 색은 `catColorArr`(약 339줄, 6색)에서 순환 사용하므로 7개 이상이어도 자동으로 색이 돌아간다. 색을 더 예쁘게 하려면 `catColorArr`에 oklch 색을 추가하면 된다.
- **카드 추가/삭제** → `cards` 배열만 수정. 배열 첫 번째가 기본 선택값이다(현재 기본 `CSP(WS)`).
- **사용자 추가/변경** → `users` 배열 수정. (지금은 2명 전제로 UI가 짜여 있으니 3명 이상으로 늘리면 사용자별 통계 영역만 가볍게 확인해라.)

이 세 배열만 바꾸면 입력 드롭다운·칩·통계·색이 전부 따라 바뀐다. **다른 곳 손댈 필요 없음.**

### 입력 필드 자체를 바꾸고 싶을 때
- 한 건의 지출 데이터 모양(entry)은 `addEntry()`에서 만든다:
  `{ id, date, amount, category, user, card, memo }`
- 필드를 추가하려면 ① `state`의 `draft` 초기값에 필드 추가 ② `addEntry()`의 entry 객체에 추가 ③ 입력 UI(템플릿의 좌측 입력 영역/모바일 입력 카드) 추가 ④ 시트 백엔드(`Code.gs`의 `HEADER` 배열과 `addEntry`/`readAll`)에도 같은 필드 추가.
- 시트 컬럼과 앱 필드는 **짝을 맞춰야** 한다. `Code.gs`의 `var HEADER = ['id','date','amount','category','user','card','memo'];` 가 시트 컬럼 순서다.

## 3. 데이터/동기화 동작 (알아둘 것)

- 앱은 켜질 때 `pull()`로 시트 전체를 GET 해서 로컬(localStorage `fam-budget-v1`)에 캐시한다.
- 입력/삭제/예산변경은 화면에 즉시 반영(낙관적)하고 `send()`로 시트에 POST 저장한다.
  - POST는 `Content-Type: text/plain` 으로 보낸다. **바꾸지 마라** — CORS 프리플라이트를 피하려는 의도다.
- 시트 URL은 `state`의 `DEFAULT_URL` 상수에 박혀 있다(약 209줄). 시트를 새로 배포해 URL이 바뀌면 이 값만 교체.
- 달별로 시트 탭(`2026-07`…)이 자동 생성된다. 예산은 `_meta` 탭.
- 네트워크 실패 시 로컬로 폴백하고 우측 상단 배지에 "저장 실패/연결 실패" 표시.

## 4. 변경 후 검증

로컬에서 그냥 파일 열면 CORS로 fetch가 막힐 수 있으니 간단한 정적 서버로 확인해라:

```bash
cd <이 폴더>
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000
```

- 앱이 뜨고 우측 상단 배지가 "● 시트 연결됨"(초록)이 되는지 확인.
- 항목/카드를 바꿨다면 입력 드롭다운·칩에 반영됐는지, 실제로 추가하면 시트에 줄이 쌓이는지 확인.
- 콘솔 에러 없는지 확인.

## 5. 배포 (GitHub Pages)

1. 새 저장소 생성 (예: `family-budget`).
2. `index.html`, `support.js`, `apps-script/`(참고용) 전부 커밋·푸시. **`index.html`과 `support.js`는 반드시 같은 폴더(루트)에** 둔다.
3. Settings → Pages → Source: *Deploy from a branch* → Branch: `main` / `(root)` → Save.
4. 1~2분 후 `https://<사용자명>.github.io/family-budget/` 로 접속.
5. 이후 변경은 `index.html` 고쳐서 커밋·푸시하면 자동 반영.

---

## 6. 하지 말 것
- `support.js` 편집 금지.
- 프레임워크(React CLI, Vite 등)로 재작성 금지 — 이 파일은 자체 런타임으로 동작한다.
- POST의 `text/plain` → `application/json` 변경 금지(시트 저장이 깨진다).
- UI 색/레이아웃/폰트는 요청 없으면 건드리지 마라. 요청받은 것만 최소로 변경.

## 7. 앱 개요 (맥락)
- 입력: 금액($, USD), 날짜, 항목, 사용자, 카드, 메모.
- 전체 예산 및 항목별 예산 대비 **남은 금액을 게이지**로, **일자별 지출 추이**를 막대로 표시.
- 데스크톱은 좌(입력)/우(요약) 두 칸, 모바일은 세로 흐름으로 자동 전환. 상단 홈/대시보드 탭, 자동/PC/모바일 미리보기 토글.
